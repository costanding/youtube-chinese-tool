# YouTube 汉化工具 - 技术架构文档

> 本文档供 AI 或开发者快速理解整个系统，接手维护。

## 项目概述

一个 YouTube 视频汉化桌面工具，功能：
- 下载 YouTube 视频/音频/字幕
- 将英文视频自动汉化为中文（语音+字幕）
- 桌面 GUI 一键操作

## 技术栈

| 层级 | 技术 | 文件 |
|------|------|------|
| GUI | PowerShell + WinForms | launcher.ps1 |
| 汉化核心 | Python 3.10+ | bin/dubber.py |
| YouTube 下载 | yt-dlp（独立 exe） | bin/yt-dlp.exe |
| 音视频处理 | ffmpeg（独立 exe） | bin/ffmpeg.exe, ffprobe.exe |
| 语音合成 | edge-tts（微软 Edge TTS） | Python pip 包 |
| 翻译 | deep-translator（Google Translate） | Python pip 包 |

## 核心流程详解

### 1. 下载阶段 (dubber.py: download)

```
输入: YouTube URL
输出: (标题, 视频路径, 字幕路径)

流程:
1. yt-dlp --print title 获取视频标题
2. 检查视频是否已存在（跳过重复下载）
3. yt-dlp 下载视频（H.264 编码，兼容性最好）
   - 格式: bestvideo[ext=mp4][vcodec^=avc1]+bestaudio[ext=m4a]
4. 优先下载中文字幕（zh-Hans, zh）
   - 如果没有中文，下载英文字幕
   - 绝不用日文字幕（避免二次翻译质量损失）
5. 返回文件路径
```

### 2. 翻译阶段 (dubber.py: translate)

```
输入: [(start, end, text), ...] 字幕条目
输出: [(start, end, zh_text), ...] 翻译后的条目

流程:
1. 检测字幕是否已是中文（正则检测中文字符）
   - 是 → 跳过翻译，直接返回
   - 否 → 继续翻译
2. 按 30 条一批翻译（保留上下文）
3. 翻译引擎优先级:
   - deep-translator (Google Translate) - 重试 3 次
   - MyMemory API - 兜底
   - 原文 - 最终兜底
4. 用句号连接原文，翻译后按句号分割（保持语义流）
```

### 3. 语音合成阶段 (dubber.py: gen_voice_per_sentence)

```
输入: [(start, end, zh_text), ...] 中文字幕条目
输出: [(start, end, audio_path), ...] 语音文件路径

流程:
1. 逐句生成 TTS（不是合并成大段）
2. 语速初始 +25%（中文比英文慢）
3. 检查实际时长 vs 目标时长:
   - 如果 TTS > 目标 * 1.3 → 用 ffmpeg atempo 压缩
   - 如果 TTS < 目标 * 0.7 → 重新生成（减速）
4. 并发 15 路生成（asyncio.Semaphore）
5. 清理文本中的特殊字符（♪▶◀等）
6. 失败时自动重试（默认语速）
```

### 4. 合成阶段 (dubber.py: merge_dub)

```
输入: 视频路径, TTS片段列表
输出: .zh.mp4 最终视频

流程:
1. 提取原始音频（44100Hz 立体声）
2. 逐句拼接中文语音时间轴:
   - 计算每句间隔（gap）
   - 插入静音填充间隔
   - 如果 TTS 太长，用 atempo 压缩
3. 背景音压低到 8%（volume=0.08）
4. 混合: 背景音 + 中文语音（amix）
5. 合成最终视频:
   - 视频流直接复制（-c:v copy，不重编码）
   - 音频用 AAC 编码（-c:a aac -b:a 192k）
   - 时长截断到原视频长度（-t duration）
6. 输出到临时文件，再复制到最终位置（避免覆盖原视频）
```

## GUI 架构 (launcher.ps1)

### 技术方案
- PowerShell + System.Windows.Forms（WinForms）
- 纯原生，无第三方依赖
- 支持中文界面（UTF-8 BOM 编码）

### 控件结构
```
Form (窗口)
├─ Label "YouTube 汉化工具"
├─ TextBox (URL输入)
├─ ComboBox (语音选择: 男声/女声/男声2/女声2)
├─ ComboBox (模式选择: 配音+背景音/仅字幕嵌入/仅生成音频/仅字幕文件)
├─ Button × 7 (汉化/下载视频/下载音频/下载字幕/视频信息/打开目录/更新内核)
├─ Label (状态栏)
└─ Label (路径提示)
```

### 依赖检测
首次启动时自动检测:
1. Python 是否安装 → 没有则用 winget 自动安装
2. pip 包是否安装 → 没有则自动 pip install
3. yt-dlp 是否存在 → 没有则提示下载

### 汉化按钮事件流
```
点击「汉化视频」
  → 获取 URL 和选项
  → Start-Process python.exe dubber.py URL --voice XX --mode XX
  → 新开控制台窗口运行（不阻塞 GUI）
  → 状态栏显示「汉化已启动」
```

## 关键设计决策

### 1. 为什么用 yt-dlp 独立 exe？
- 不需要 Python 环境也能下载
- 自带更新功能（-U）
- 社区活跃，YouTube 接口变化时快速修复

### 2. 为什么用 ffmpeg 独立 exe？
- yt-dlp 下载的视频需要合并音视频流
- 音频格式转换（webm → mp3）
- 语音时长调整（atempo）
- 背景音混合（amix）

### 3. 为什么逐句 TTS 而不是整段？
- 精确对齐时间轴（每句话对应原始时间点）
- 避免一句话太长导致后面全部错位
- 可以单独调整每句语速

### 4. 为什么用 Edge TTS 而不是其他？
- 免费，无需 API Key
- 中文语音质量好（微软神经网络语音）
- 支持多种声音（男声/女声）
- 支持 rate 参数调整语速

### 5. 为什么优先用 YouTube 中文字幕？
- 人工翻译质量远超机器翻译
- 时间轴天然对齐
- 不需要额外翻译步骤

### 6. 为什么背景音只保留 8%？
- 中文语音是主体，需要清晰
- 完全去掉背景音会显得突兀
- 8% 保留氛围感，不干扰语音

## 已知问题和限制

### yt-dlp 相关
- YouTube 经常改接口，需要定期更新
- 部分视频有地区限制/年龄限制
- 字幕下载可能被 429 限流

### 翻译相关
- Google Translate 质量有限，不如人工
- 30 条/批的上下文窗口可能丢失跨批语境
- 特殊术语可能翻译不准

### TTS 相关
- Edge TTS 需要联网（微软云端服务）
- 生成的语音偏机械，不如真人
- 部分特殊字符会导致生成失败
- 语速调整可能有微小偏差

### ffmpeg 相关
- 视频流直接复制，不重编码（速度快但不能改分辨率）
- 输出文件可能比原文件大（音频码率 192k）
- 合成时临时文件较大（需要足够磁盘空间）
